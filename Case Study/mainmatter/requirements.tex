\chapter{Requirements Analysis}
\label{chapter:requirements}
% ============================================================

This chapter applies TOGAF ADM Phase B (Business Architecture) to derive requirements from the enterprise context, stakeholder concerns and identified risks defined in Chapter~\ref{chapter:enterprise-context}. Requirements are not generated from technical preferences; each requirement is explicitly traced to a source.

\subsection{Performance}

\begin{itemize}
    \item \textbf{NFR-PERF-01:} Customer portal page load time $\leq$ 2 seconds under normal load (1,200 transactions/day). Source: UR-06.
    \item \textbf{NFR-PERF-02:} Customer portal page load time $\leq$ 5 seconds under peak load (10,000 transactions/day). The web cluster auto-scaling must absorb the 8:1 peak-to-average ratio.
    \item \textbf{NFR-PERF-03:} RabbitMQ message processing latency $\leq$ 500 ms under normal load. Source: FR-03.
    \item \textbf{NFR-PERF-04:} Database query response time $\leq$ 200 ms for 95\% of PostgreSQL queries under normal conditions. Source: FR-01.
    \item \textbf{NFR-PERF-05:} WAN failover (MPLS $\rightarrow$ IPSec VPN) must complete within 60 seconds without manual intervention.
\end{itemize}

\subsection{Security}

\begin{itemize}
    \item \textbf{NFR-SEC-01:} All data in transit between VLANs, sites and external parties must be encrypted (TLS 1.2 minimum; TLS 1.3 preferred). Source: legal / GDPR.
    \item \textbf{NFR-SEC-02:} All data at rest in the HR/Payroll and Accounting clusters must be encrypted using AES-256 or equivalent. Source: UR-04, GDPR.
    \item \textbf{NFR-SEC-03:} Multi-factor authentication (MFA) must be enforced for all IT staff, management and Finance/HR roles via the IAM. Source: IT Director, risk profile.
    \item \textbf{NFR-SEC-04:} Firewall rule changes must be logged, version-controlled and require dual approval from IT management. Source: MR-05.
    \item \textbf{NFR-SEC-05:} Security logs must be retained for a minimum of 12 months online and 36 months in cold storage. Source: GDPR Article 5(1)(e), incident response requirements.
\end{itemize}

\subsection{Scalability and Capacity}

\begin{itemize}
    \item \textbf{NFR-CAP-01:} The web cluster must auto-scale horizontally to handle the 8:1 peak-to-average load ratio without manual intervention.
    \item \textbf{NFR-CAP-02:} MongoDB must be capable of horizontal shard expansion to accommodate customer base growth of up to 300\% without architectural redesign.
    \item \textbf{NFR-CAP-03:} Storage capacity for all systems must include a minimum 30\% headroom above projected 12-month utilisation.
    \item \textbf{NFR-CAP-04:} Capacity metrics must trigger a planning review alert when any critical resource exceeds 70\% of provisioned capacity sustained over 7 days.
\end{itemize}
